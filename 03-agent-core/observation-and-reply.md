# Observation And Reply

很多初学者只注意到了“调用工具”，却忽略了后面还有两步：

- observation
- reply

这两步其实非常关键。

## 什么是 `observation`

工具执行完之后，`Agent` 需要读取它返回的结果。

比如：

- `get_time` 返回当前时间字符串
- `save_note` 返回“保存成功”
- `read_notes` 返回笔记内容

这个“读取工具结果”的过程，就是 `observation`。

## 为什么不能跳过 `observation`

如果不读取结果，程序就不知道刚才的行动到底发生了什么。

也就是说，`act` 之后如果没有 `observe`，系统很容易变成：

- 做了事
- 但不知道结果
- 然后胡乱回复

## 什么是 `reply`

`reply` 是把工具结果变成对用户有用的话。

例如工具返回：

`2026-04-04 15:30:00`

直接把这串值扔给用户虽然也可以，但更像机器。

而 `reply` 会把它整理成：

`The current time is 2026-04-04 15:30:00`

## 为什么这一步对学习很重要

因为它能让你意识到：

- 工具结果不等于最终回答
- `Agent` 的工作不是只会执行动作
- 它还要把动作结果转化成可理解输出

这也是很多真实 `Agent system` 设计里的关键一环。
