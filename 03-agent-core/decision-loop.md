# Decision Loop

`Agent` 和普通脚本的一个重要区别，是它不会每次都固定做同一件事。

它需要先判断：

“这次应该调用哪个工具？”

这就是最简单的 `decision loop`。

## 一个最朴素的判断方式

在你的第一个项目里，我们不会上来就用模型做决策。

我们先用规则判断：

```python
if "time" in text:
    return "get_time"
```

这不是最强的方法，但对初学者非常好，因为它有两个优点：

- 你能完全看懂
- 你能亲眼看到“决策”是怎么发生的

## 为什么这也算 `Agent`

很多初学者会误以为：

“只有接入大模型、自动推理、很聪明，才叫 Agent。”

其实不是。

只要系统具备下面这个最小结构：

- 接收输入
- 判断该做什么
- 执行动作
- 读取结果
- 返回结果

它就已经具备了最基础的 `Agent` 形态。

你现在这个学习项目，属于：

- 有 `Agent` 基本结构
- 但决策逻辑还是规则驱动

所以它更准确地说，是一个 `rule-based agent`，或者说一个最小 `agent loop` 教学版本。

它的目的不是一上来就做“完整现代 AI agent”，而是先让你把结构看清楚。

## 以后会怎么升级

你现在先学规则判断。

以后你会看到更复杂的版本，比如：

- 用 `LLM` 判断该调用哪个工具
- 一次决定多个动作
- 根据上一步结果继续规划下一步

但这些升级版，都是从这个最简单的 `decision loop` 长出来的。

所以不要把“有没有 `LLM`”和“是不是 `Agent`”机械地绑死。

更准确的理解是：

- 没有 `LLM`，也可以有广义的 `Agent`
- 有了 `LLM`，就更接近今天常说的 `AI Agent`
