# Upgrade Path

这章的目标不是马上把你拖进复杂实现，而是让你知道下一步应该怎么走。

## 从当前项目升级到 LLM-Based Agent 的最自然路线

### 第一步：保留现有工具

先不要急着改 `tools.py`。

你现在已经有了：

- `get_time`
- `save_note`
- `read_notes`

这些工具本身没有问题，可以继续用。

## 第二步：只替换“决策层”

你真正先要替换的是 [app.py](/C:/Users/admin/Desktop/MyAgent/05-my-first-agent/my-first-agent/app.py) 里那部分写死的规则判断。

也就是把：

- `if "time" in text`
- `if text.startswith("save ")`

这些规则，逐渐升级成：

- 调用 `LLM`
- 让 `LLM` 判断该调用哪个工具
- 让 `LLM` 提取参数

## 第三步：保留程序执行工具

即使接入了 `LLM`，真正执行工具的也还是你的程序。

所以结构上会变成：

1. 用户输入
2. 程序把工具信息提供给模型
3. 模型返回想调用的工具和参数
4. 程序执行工具
5. 程序把结果交回模型
6. 模型给出最终回复

## 第四步：最后再考虑更复杂的能力

等你把最基础的 `LLM tool calling` 跑通之后，再考虑：

- 多轮调用
- 简单 `Memory`
- 多工具连续规划
- 更复杂的任务分解

## 我建议你的下一步

最自然的路线是：

1. 先彻底跑通当前规则型项目
2. 再看这一章
3. 然后做一个“只替换决策层”的升级版

这样你会比一上来直接套框架，理解得深很多。
