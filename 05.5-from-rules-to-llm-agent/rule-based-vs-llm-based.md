# Rule-Based Vs LLM-Based

你现在已经做出来的版本，更准确地说是：

`rule-based agent`

它已经有 `Agent` 的最小结构，但还没有把“决策”交给 `LLM`。

## `rule-based agent` 在做什么

它的特点是：

- 输入进来
- 程序用你写死的规则判断
- 决定调用哪个工具
- 返回结果

例如：

```python
if "time" in text:
    return "get_time"
```

优点是：

- 容易理解
- 可预测
- 适合零基础学习结构

缺点是：

- 很死
- 能力边界明显
- 你得提前把规则全写出来

## `LLM-based agent` 在做什么

它的核心变化是：

“决策”这一步不再完全靠手写规则，而是交给 `LLM`。

也就是说，系统会变成：

- 用户输入目标
- `LLM` 理解目标
- `LLM` 判断是否该调用工具
- `LLM` 选择工具和参数
- 工具执行后把结果返回
- `LLM` 再基于结果给出最终回复，或者决定下一步

## 两者最本质的差别

不是“有没有代码”，也不是“是不是更高级”。

最本质的差别在于：

- `rule-based agent` 的决策策略是程序员提前写死的
- `LLM-based agent` 的决策策略有一部分交给了模型

## 为什么我们先学规则型

因为如果一上来就接 `LLM`，你会同时面对这些问题：

- API
- key
- token
- prompt
- tool schema
- 模型输出格式

这对零基础太容易把结构感冲散。

所以我们先学规则型，不是因为它更“标准”，而是因为它更适合当教学起点。
